 Classes

The code you write should have the following qualities:

  • Transparent: The consequences of change should be obvious in the code that is changing in distant code that relies upon it.
  • Reasonable: The cost of any change should be proportional to the benefits in change achieves.
  • Usable: Existing code shold be usable in new and unexpectd contexts.
  • Exemplary: The code itself should encourage those who change it to perpetuate these qualities

Code that is Transparent, Reasonable, Usable, and Exemplary (TRUE) not only meet today's needs but can also be changed to meet the needs of the future. The first step in creating code that is TRUE is to ensure that each class as a single, well-defined responsibility. (above)

In a class-based OO language like Ruby, methods are defined in classes. The classes you create will affect how you think about your application forever. They define a virtual world, one that constrains the imagination of everyone downstream. You are constructing a box that may be difficult to think outside of.

Applications that are easy to change consist of classes that are easy to reuse. Reusable classes are pluggable units of well-defined behavior that have few entanglements. Thus, a class should do the smallest possible useful thing; that is, it should have a single responsibility.

Source: Practical Object-Oriented Design in Ruby
